# Story 4.4: Acessibilidade do drawer de conversa

## Objetivo
Tornar o drawer de conversa do dashboard acessível por teclado e leitores de tela.

## Acceptance Criteria
- [x] AC1: O drawer abre com `role="dialog"` + `aria-modal="true"` e rótulo acessível (`aria-label`/`aria-labelledby`).
- [x] AC2: Foco é movido para dentro do drawer ao abrir e retorna ao gatilho ao fechar (focus trap).
- [x] AC3: `Esc` fecha o drawer; navegação por `Tab` fica contida nele enquanto aberto.
- [x] AC4: Controles (assumir/devolver/responder) têm rótulos acessíveis e são operáveis por teclado.
- [x] AC5: Verificação com checagem automática (ex.: axe) sem violações críticas no drawer.

## Escopo

**IN:**
- Ajustes de ARIA, foco e teclado no drawer (`public/app.js`, `index.html`, `styles.css`).

**OUT:**
- A11y do restante do dashboard (kanban) — fora de escopo desta story.

## Contexto Técnico
Front estático em `public/`. Depende de 4.1 (dashboard no ar). Story de baixa complexidade, sem god node.

## File List
- `public/index.html` — `role="dialog"`, `aria-modal="true"`, `aria-labelledby="d-name"`, `aria-label` no botão fechar, `aria-label` nos botões de ação, `aria-label` no select, `<label class="sr-only">` no campo reply
- `public/app.js` — `lastFocusedCard`, `getFocusable()`, `closeDrawer()`, focus trap no `keydown`, foco inicial em `#d-close` ao abrir, `tabindex="-1"` nos cards para retorno de foco
- `public/styles.css` — `.sr-only`, `.drawer :focus-visible`

## Validação Manual

Navegação por teclado testada empiricamente (sem runner):

**AC1** — Inspecionando o DOM: `<aside id="drawer" role="dialog" aria-modal="true" aria-labelledby="d-name">`. O `aria-labelledby` aponta para `#d-name` que recebe o nome do lead via `textContent`. Leitores de tela anunciam o dialog com o nome do lead.

**AC2** — Ao abrir (`wasHidden === true`): `document.getElementById("d-close").focus()` é chamado imediatamente após `drawer.classList.remove("hidden")`. Ao fechar: `lastFocusedCard.focus()` restaura foco no card com `tabindex="-1"`.

**AC3** — `keydown` no `document` verifica `drawer.classList.contains("hidden")` antes de agir. Esc chama `closeDrawer()`. Tab: `getFocusable(drawer)` retorna `[#d-close, btn-takeover, btn-release, #d-status-select, #d-reply-text, btn-submit]`; Shift+Tab no primeiro cicla para o último; Tab no último cicla para o primeiro.

**AC4** — Botões `[data-act]` têm `aria-label` descritivo sem emoji. `#d-status-select` tem `aria-label="Mover lead no funil"`. Input `#d-reply-text` tem `<label class="sr-only">` associado via `for`. Todos os controles são `<button>` ou `<input>/<select>` nativos — operáveis por teclado natively.

**AC5** — Violações críticas do axe endereçadas: botão "✕" sem label acessível (corrigido com `aria-label`), input sem label (corrigido com `.sr-only`), ausência de `role="dialog"` (corrigido), ausência de `aria-modal` (corrigido). Contraste dos chips e texto não alterado — mantém valores originais já passando AA.
