# Story 5.4: Módulo CRM/kanban no portal

## Objetivo
Portar o kanban + drawer de conversa existentes para um módulo `/crm` do portal Next.js, usando o design system e os endpoints de leads atuais.

## Acceptance Criteria
- [x] AC1: Rota `/crm` renderiza o kanban (colunas por estágio do funil) consumindo os endpoints de leads existentes.
- [x] AC2: Drawer de conversa funcional (ver histórico, assumir/devolver IA, mover funil, responder como humano, editar) — paridade com o dashboard atual.
- [x] AC3: Aplica o design system (5.3): estados de loading/empty/erro, responsivo, a11y do drawer.
- [ ] AC4: Distinção visual IA vs humano nas mensagens (consome a [[4.3-distinguir-ia-vs-humano]]). **PENDENTE: campo 'author' ausente na tabela messages — balões 'out' não distinguem IA vs humano. Awaiting Story 4.3.**
- [ ] AC5: Respeita RBAC (5.2): `viewer` vê mas não edita/assume. **PENDENTE: Story 5.2 não concluída.**

## Escopo

**IN:**
- Migrar kanban + drawer para o módulo Next, com paridade funcional e UX melhorada.

**OUT:**
- Lista rica de leads (Story 5.5 — vista separada).
- Realtime (pode entrar como melhoria posterior).

## Contexto Técnico
Depende de 5.1 (shell) e 5.3 (design system); usa 4.3 (autoria das mensagens) e 5.2 (RBAC). Reaproveita `GET /api/leads`, `/api/leads/:id` e as ações de `routes/api.ts` (como route handlers ou funções existentes). Catálogo: [[../../agents/ux/components]].
