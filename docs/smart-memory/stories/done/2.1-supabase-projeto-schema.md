# Story 2.1: Criar projeto Supabase e aplicar schema

## Objetivo
Provisionar o Postgres de produção no Supabase com o schema completo, incluindo os ajustes de hardening da Wave 0 (coluna `external_id`, constraints).

## Acceptance Criteria
- [ ] AC1: Projeto Supabase criado; `supabase/schema.sql` aplicado com sucesso (tabelas `leads`, `messages`, ENUM `lead_status`, índices `idx_messages_lead`, `idx_leads_status`, `idx_leads_followup`).
- [ ] AC2: `messages.external_id text` + índice único parcial `idx_messages_external_id ... WHERE external_id IS NOT NULL` presentes (incorpora Story 1.1). ✅ *no schema.sql*
- [ ] AC3: Constraint `chk_followup CHECK (follow_up_count >= 0)` aplicada (gap #7 do schema). ✅ *no schema.sql*
- [ ] AC4: `id` de `leads` e `messages` são `uuid DEFAULT gen_random_uuid()`; FK `messages.lead_id → leads.id ON DELETE CASCADE`. ✅ *no schema.sql*
- [ ] AC5: Trigger `trg_leads_updated_at` ativo; `INSERT` de teste em `leads` popula `created_at`/`updated_at` e `UPDATE` atualiza `updated_at`. ✅ *no schema.sql*
- [ ] AC6: RLS permanece **desabilitado** nesta story (acesso por service_role); habilitação fica na Story 4.2. ✅ *comentado no schema.sql*

> AC2–AC6 estão no schema.sql. AC1 requer credenciais do usuário (ver bloqueio abaixo).

## BLOQUEIO — aguardando credenciais do usuário

Para desbloquear, o usuário precisa:

1. **Criar projeto Supabase** em [supabase.com](https://supabase.com) (se ainda não existe).
2. No painel do projeto → **Settings → API**, copiar:
   - `Project URL` → `SUPABASE_URL`
   - `service_role` key (aba "Project API keys") → `SUPABASE_SERVICE_ROLE_KEY`
3. Adicionar ao `.env` local:
   ```
   SUPABASE_URL=https://<project-ref>.supabase.co
   SUPABASE_SERVICE_ROLE_KEY=eyJ...
   ```
4. Aplicar o schema em uma destas formas:
   - **SQL Editor do Supabase:** colar o conteúdo de `supabase/schema.sql` e executar.
   - **CLI local:** `npx supabase db push` (requer `supabase link --project-ref <ref>`).

Após aplicar, o agente valida com smoke tests (ver abaixo).

## Smoke tests (pós-aplicação)

```sql
-- AC1/AC4: tabelas e UUID
SELECT id, phone, status FROM leads LIMIT 1;

-- AC2: external_id existe e índice parcial está ativo
SELECT indexname, indexdef FROM pg_indexes
  WHERE tablename = 'messages' AND indexname = 'idx_messages_external_id';

-- AC3: constraint chk_followup
SELECT conname FROM pg_constraint
  WHERE conrelid = 'leads'::regclass AND conname = 'chk_followup';

-- AC5: trigger ativo
SELECT tgname FROM pg_trigger
  WHERE tgrelid = 'leads'::regclass AND tgname = 'trg_leads_updated_at';

-- AC5: teste funcional
INSERT INTO leads (phone) VALUES ('+5511999990001');
SELECT created_at, updated_at FROM leads WHERE phone = '+5511999990001';
UPDATE leads SET name = 'Teste' WHERE phone = '+5511999990001';
SELECT created_at, updated_at FROM leads WHERE phone = '+5511999990001';
-- updated_at deve ser > created_at após o UPDATE
DELETE FROM leads WHERE phone = '+5511999990001';
```
