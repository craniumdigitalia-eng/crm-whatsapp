# Story 5.3: Design system branded do portal (KV)

## Objetivo
Definir o design system do portal (tokens, componentes base, layout branded) a partir do KV / identidade visual da Cranium, aplicando-o ao shell e aos módulos.

## Acceptance Criteria
- [ ] AC1: Tokens de design (cores, tipografia, espaçamento, raios, sombras) derivados do KV, centralizados (tema).
- [ ] AC2: Componentes base do portal (botão, input, select, tabela, card, badge, drawer/modal, navegação) padronizados e reutilizáveis.
- [ ] AC3: Shell (sidebar + topbar) aplica o branding; estados de loading/empty/erro padronizados (resolve gaps de UX: sem skeleton, sem empty-state).
- [ ] AC4: Responsivo com breakpoints (resolve o gap "mobile quebrado < 440px" do catálogo UX).
- [ ] AC5: A11y base: foco visível, contraste AA, drawer/modal acessível (absorve a antiga Story 4.4); checagem axe sem violações críticas.

## Escopo

**IN:**
- Tema/tokens, biblioteca de componentes base, padrões de estado, responsividade e a11y base.

**OUT:**
- Implementação funcional de cada módulo (consome esta base).
- Lógica de dados.

## Contexto Técnico
Depende de 5.1 (shell). **Bloqueado pela entrega do KV pelo usuário** (pendência registrada no shared-context). Absorve a story [[4.6-redesign-visual-dashboard]] e a a11y da [[4.4-a11y-drawer]]. Base do catálogo: [[../../agents/ux/components]].
