# Story 5.7: Módulo de Agendamento de reuniões

## Objetivo
Quando um lead é qualificado, disparar o agendamento de reunião **via Google Calendar API direto** (sem Make) e notificar a equipe; expor a gestão de reuniões no portal.

## Acceptance Criteria
- [ ] AC1: Gatilho de domínio: ao lead transitar para `qualificado`, o sistema cria/oferece um evento no **Google Calendar** (via Google Calendar API/SDK direto) — gatilho confiável e idempotente (não duplica por reprocesso).
- [ ] AC2: Notificação à equipe com resumo do lead (e-mail/convite do próprio evento Google, ou WhatsApp interno via Evolution) — **sem Make**.
- [ ] AC3: Rota `/agenda` lista reuniões agendadas (lead, horário, status) e permite ações básicas (cancelar/reagendar) refletidas no Google Calendar.
- [ ] AC4: Persistência das reuniões (tabela `meetings` — coordenar com a crm-data) ligada ao lead, espelhando o id do evento Google.
- [ ] AC5: Integração resiliente: falha do Google Calendar não quebra o fluxo de qualificação (degradação graciosa + retry/log).
- [ ] AC6: OAuth do Google conectado uma vez pelo usuário (princípio: minimizar esforço do usuário); RBAC (5.2) e design system (5.3) aplicados.

## Escopo

**IN:**
- Gatilho na qualificação, integração **direta** com Google Calendar API (OAuth + criar/cancelar/reagendar evento), notificação, vista de agenda, persistência.

**OUT:**
- Make / Cal.com (descartados — ver ADR-004).
- Disponibilidade/round-robin avançado de calendários (v2).
- O canal de mensagens em si (Evolution — stories 5.8/5.9).

## Contexto Técnico
**God node:** o gatilho de qualificação toca o caminho do agente/handler (`agent/agent.ts` `applyTool` status→`qualificado`, e/ou `crm/leads.ts`). Preferir um **evento de domínio** desacoplado a acoplar I/O de calendário dentro do loop do agente. Depende de 5.3. **Google Calendar API direto** (há SDK/MCP disponível) substitui o Make — menos um terceiro para o usuário configurar/pagar (ver [[../../decisions/ADR-004-canal-whatsapp-qr-vs-make]]). Ver [[project/modules]] para a fronteira raciocínio×execução.

## Acceptance Criteria — Frontend (AC3 / AC6)

- [x] AC3: Rota `/agenda` lista reuniões via GET /api/agenda/events; permite criar, editar e excluir eventos, refletindo no Google Calendar.
- [x] AC6: Design system (5.3) aplicado — paleta brand-purple, tokens CSS, responsivo.
- [x] Acessibilidade: aria-label em todos os botões/células, role="grid", role="dialog", focus-visible.
- [x] Estados: loading (ícone girando), erro com mensagem específica quando Calendar não conectado, estado vazio.
- [x] Visão Mês + Semana com toggle, navegação anterior/hoje/próximo.
- [x] Modal criar (clicar em dia/horário) e editar/excluir (clicar em evento).

- [x] Link do Google Meet exibido quando disponível (edição).
- [x] Build `npm run build` passando sem erros.
