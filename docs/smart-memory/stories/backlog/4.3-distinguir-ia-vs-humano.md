# Story 4.3: Distinguir mensagem de IA vs humano

## Objetivo
Diferenciar, em `messages`, se uma mensagem `out` foi enviada pelo agente de IA ou por um humano, exibindo essa distinção no dashboard.

## Acceptance Criteria
- [ ] AC1: `Message` (`src/types.ts`) e a tabela `messages` ganham um campo de autoria (ex.: `author text CHECK ('ia','humano','sistema')` ou `sender_type`), default coerente.
- [ ] AC2: `addMessage` passa a registrar a autoria; respostas do agente gravam `'ia'`, reply manual via `/api/leads/:id/reply` grava `'humano'`, follow-up automático grava `'sistema'` (ou `'ia'`, decidir no ADR).
- [ ] AC3: O dashboard exibe visualmente a origem de cada mensagem `out`.
- [ ] AC4: Mensagens antigas sem autoria recebem default retroativo sem quebrar a UI.
- [ ] AC5: `npm run build` compila; teste cobre as três origens.

## Escopo

**IN:**
- Campo de autoria em tipos + schema + persistência + UI.

**OUT:**
- Rastreio de custo/tokens (gap #6 — pode virar story 4.5 ou própria).

## Contexto Técnico
God node: `types.ts` (e ajuste em `crm/leads.ts`/`addMessage`). Schema é autoridade da crm-data; coordenar a coluna. Hoje IA e humano são ambos `direction='out'` indistinguíveis. Depende de 4.1.
