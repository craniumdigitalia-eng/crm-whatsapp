# Story 9.2: Convite de usuários por cliente

## User Story
Como **admin de um cliente** quero **convidar outros usuários da minha equipe (atendentes) para a minha conta** para **operar o CRM com o time todo sem pedir para a Cranium criar cada login**.

## Objetivo
Adicionar o fluxo de convite multi-usuário dentro da conta do cliente, reusando os papéis admin/atendente que já existem (story 5.2), sem depender da Cranium.

## Acceptance Criteria
- [ ] AC1: Um admin do cliente convida um usuário por e-mail; o convidado recebe convite e cria acesso (via Supabase Auth) vinculado à **mesma conta/tenant**.
- [ ] AC2: O convite atribui um **papel** (admin/atendente) respeitando o RBAC existente (story 5.2); atendente não vê o que só admin pode ver.
- [ ] AC3: Admin gerencia usuários da conta: listar, alterar papel, revogar acesso.
- [ ] AC4: O número de usuários respeita a **cota do plano** ([[9.1-limites-por-plano]]); estourar bloqueia novo convite com aviso de upgrade.
- [ ] AC5: Convite expira e é reenviável; convite pendente não concede acesso.
- [ ] AC6: Nenhum usuário de um cliente enxerga dados de outro cliente (isolamento por tenant preservado).

## Escopo

**IN:**
- Fluxo de convite por e-mail, atribuição de papel, gestão de usuários da conta, expiração/reenvio, respeito à cota.

**OUT:**
- Criação do papel super-admin (cross-cliente) — [[6.3-painel-super-admin]].
- Cotas de plano em si — [[9.1-limites-por-plano]].

## Dependências
- Depende do RBAC de [[../active/5.2-auth-rbac-interno]] (papéis já existem).
- Cota de usuários relaciona [[9.1-limites-por-plano]].

## Notas de risco
- Isolamento por tenant é crítico: um convite não pode dar acesso cruzado entre clientes. No modelo "deploy por cliente" o isolamento é natural (deploy separado); no banco compartilhado ([[10.1-multi-tenant-db-compartilhado]]) depende de RLS por `org_id`.

## Contexto Técnico
Auth = Supabase Auth SSR + RBAC (story 5.2), papéis admin/atendente, migrations 004/006 (auth/profiles, role-lock). Convite reusa o fluxo de convite do Supabase Auth quando aplicável.
