# Story 5.11: Guia de setup (Evolution + Meta Lead Ads + Google Calendar)

## Objetivo
Documento passo-a-passo enxuto para o usuário ligar tudo com o **mínimo esforço**: escanear 1 QR (WhatsApp), conectar o Google Calendar e configurar o anúncio de **formulário instantâneo (Meta Lead Ads)**.

## Acceptance Criteria
- [ ] AC1: Passo **WhatsApp:** abrir `/whatsapp` no portal → escanear o QR → número conectado (nós hospedamos a Evolution; o usuário só escaneia).
- [ ] AC2: Passo **Google Calendar:** autorizar o OAuth uma vez para habilitar o agendamento automático (Story 5.7).
- [ ] AC3: Passo **Meta Lead Ads:** criar o anúncio com **formulário instantâneo** no Meta Ads Manager e **conectar a página/app ao webhook `leadgen`** (verify token + permissões `leads_retrieval`) — o lead é capturado e o opener disparado automaticamente (Story 5.10).
- [ ] AC4: Checklist de verificação: form preenchido → `/api/leadgen` → lead criado → opener no WhatsApp; lead responde → agente assume; lead qualificado → evento no Google Calendar.
- [ ] AC5: Troubleshooting básico (QR/reconexão, OAuth, webhook leadgen não chega, token da página expirado, opener não enviado).

## Escopo

**IN:**
- Documento de enablement (3 passos + checklist + troubleshooting), focado em minimizar o trabalho do usuário.

**OUT:**
- Implementação de código (é doc).
- Make / Cal.com / CTWA (descartados — ADR-004).

## Contexto Técnico
Doc de enablement do canal/aquisição/agenda decididos em [[../../decisions/ADR-004-canal-whatsapp-qr-vs-make]]. Aquisição via **formulário instantâneo** (não CTWA): o ingresso é o webhook `leadgen`, não uma mensagem do lead. Complementa a ingestão da [[5.10-meta-lead-ads]]. Curta, sem god node.
