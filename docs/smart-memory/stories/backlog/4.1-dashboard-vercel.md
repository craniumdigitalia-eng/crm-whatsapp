# Story 4.1: Dashboard na Vercel

## Objetivo
Publicar o dashboard (kanban + conversas) na Vercel, consumindo as funções `/api` serverless em vez do servidor Express local.

## Acceptance Criteria
- [ ] AC1: O dashboard (`public/`) é servido pela Vercel e carrega o kanban com leads reais do Supabase.
- [ ] AC2: As chamadas do front (`app.js`) apontam para as rotas `/api` serverless (Story 3.2), não para `localhost:3000`.
- [ ] AC3: Ações do dashboard (takeover, release, status, reply manual, edit) funcionam end-to-end contra produção.
- [ ] AC4: Build/deploy reproduzível via `vercel` (sem passos manuais ocultos).

## Escopo

**IN:**
- Deploy do front estático na Vercel e fiação às rotas serverless.

**OUT:**
- Autenticação (Story 4.2).
- Distinção visual IA vs humano (Story 4.3).
- Acessibilidade do drawer (Story 4.4).

## Contexto Técnico
Depende de 3.2 (rotas serverless prontas). Decisão estático vs Next.js App Router pode virar ADR; default é estático consumindo `/api`.
