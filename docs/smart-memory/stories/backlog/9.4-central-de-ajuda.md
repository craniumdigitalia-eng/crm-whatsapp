# Story 9.4: Central de ajuda / suporte

## User Story
Como **cliente do SaaS** quero **uma central de ajuda com documentação e um canal de suporte** para **resolver dúvidas sozinho e, quando preciso, falar com o suporte sem depender de contato pessoal com a Cranium**.

## Objetivo
Reduzir o volume de suporte manual publicando docs para o cliente final e oferecendo um canal de suporte estruturado, evitando que cada cliente vire um chamado pessoal.

## Acceptance Criteria
- [ ] AC1: **Central de ajuda** acessível a partir do portal com artigos/guias para o cliente final (conectar WhatsApp, configurar agente, entender o funil, importar leads, cobrança).
- [ ] AC2: Conteúdo reusa/adapta o guia técnico existente ([[5.11-guia-setup-evolution-leadads]]) para linguagem de cliente final (não interna).
- [ ] AC3: **Canal de suporte**: um caminho claro para abrir chamado (e-mail/formulário/chat), com registro do contato.
- [ ] AC4: Busca ou navegação por categorias na central de ajuda.
- [ ] AC5: Links contextuais: telas-chave do portal (ex.: wizard, WhatsApp, agente) apontam para o artigo relevante.

## Escopo

**IN:**
- Central de ajuda (docs para cliente final) + canal de suporte + links contextuais.

**OUT:**
- Onboarding guiado in-app — [[9.3-wizard-setup-in-app]] (a central é referência, o wizard é ação).
- Docs internos de setup técnico — [[5.11-guia-setup-evolution-leadads]] (fonte a adaptar).

## Dependências
- Reusa [[5.11-guia-setup-evolution-leadads]] como base de conteúdo.
- Complementa [[9.3-wizard-setup-in-app]].

## Notas de risco
- Conteúdo desatualizado gera mais suporte, não menos — prever processo de manutenção da central.

## Contexto Técnico
Guia técnico interno existe (story 5.11). A central de ajuda pode ser páginas no próprio Next.js ou ferramenta externa; decidir conforme esforço. Sem god nodes.
