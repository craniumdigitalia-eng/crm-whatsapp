# Story 4.6 — Redesign visual do dashboard (branding Cranium)

**Como** agência usando o CRM, **quero** um dashboard bonito e com a identidade visual da Cranium,
**para** que a ferramenta passe profissionalismo e seja agradável de usar no dia a dia.

## Contexto
O dashboard atual (`public/`) é vanilla JS/CSS, tema escuro funcional mas sem branding.
Pedido do usuário (2026-06-25): "layout bonito". Usuário vai enviar o **KV (key visual)** dos sites
da Cranium como referência de identidade (paleta, tipografia, logo, vibe).

## Escopo
- IN: redesign visual do kanban + drawer de conversa + header, aplicando paleta/tipografia/logo da marca; microinterações; responsividade; manter a a11y da Story 4.4.
- OUT: mudança de stack (segue vanilla ou evolui conforme ADR-001 — Next só se justificar); lógica de negócio.

## Pendência (bloqueador de início)
Aguarda o usuário enviar o **KV / identidade visual** (cores, fontes, logo, referências).

## Aceite (rascunho — refinar com o KV)
- AC1: paleta e tipografia da marca aplicadas (tokens CSS).
- AC2: logo no header; visual coeso kanban + drawer.
- AC3: responsivo (desktop + mobile).
- AC4: a11y da 4.4 preservada (focus trap, contraste WCAG AA).
- AC5: sem regressão funcional (todas as ações do dashboard funcionando).

## Time sugerido
crm-ux (specs visuais a partir do KV) → crm-frontend (implementação). Skills: ui-ux-pro-max, sites-tailwind-design-system.
