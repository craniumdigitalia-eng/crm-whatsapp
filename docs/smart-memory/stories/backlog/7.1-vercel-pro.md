# Story 7.1: Migrar Vercel Hobby → Pro

## User Story
Como **operador do SaaS** quero **rodar na Vercel Pro** para **ter cron sub-diário e sair dos limites do Hobby, sem depender de um pinger externo (UptimeRobot) para o alerta de queda**.

## Objetivo
Subir o plano da Vercel para Pro, liberando cron com frequência menor que diária e removendo limites do Hobby.

## Acceptance Criteria
- [ ] AC1: Projeto migrado para Vercel **Pro** (billing/plan confirmado).
- [ ] AC2: Ao menos um cron passa a rodar em frequência **sub-diária** (ex.: alerta de queda da Evolution a cada N minutos), substituindo a dependência do **UptimeRobot externo** citada no changelog de 03/jul.
- [ ] AC3: A dependência do pinger externo é removida ou documentada como redundância opcional; o `evolution-health` roda nativo no Vercel Cron.
- [ ] AC4: Limites de execução (duração/concorrência) revisados e ajustados onde necessário; nenhuma função crítica estoura timeout.
- [ ] AC5: Custo do plano Pro registrado no controle de custos de infra (relaciona com [[7.3-observabilidade]]).

## Escopo

**IN:**
- Upgrade de plano, reconfiguração dos crons para frequência sub-diária, remoção/documentação do pinger externo.

**OUT:**
- Observabilidade completa (Sentry/status page/backups) — [[7.3-observabilidade]].
- WhatsApp em escala — [[7.2-whatsapp-em-escala]].

## Dependências
- Independente; habilitador de melhorias em [[7.3-observabilidade]] e alertas.

## Notas de risco
- Baixo risco técnico; é sobretudo operacional/billing. Confirmar que os schedules de cron respeitam os limites do Pro.

## Contexto Técnico
Crons já existem em `app/api/cron/*` (`followup`, `evolution-health`) protegidos por `CRON_SECRET`. No Hobby o cron é diário; o alerta de queda hoje depende de pinger externo (ver changelog 03/jul).
